Один хаб для потоков.

Stream Hub — это сервер для управления MPEG‑TS/OTT потоками: входы (UDP/HTTP/RTSP/SRT/DVB), выходы (UDP, HLS, DASH, HTTP‑TS), транскодирование через FFmpeg, failover, SoftCAM, анализ и наблюдаемость.

Быстрый старт (для первого запуска)

Ниже — простой путь “установил → запустил → добавил канал → проверил”. Без лишних терминов.

1) Установка

Выберите команду под вашу ОС. Инсталлятор ставит бинарник в /usr/local/bin/stream и подтягивает зависимости. 

# Ubuntu / Debian (быстро, готовый бинарник)
curl -fsSL https://stream.centv.ru/install.sh | sudo bash -s -- --mode binary --runtime-only
Ubuntu/Debian: скачает и установит.
# CentOS / RHEL / Rocky / Alma (совместимо, сборка из исходников)
curl -fsSL https://stream.centv.ru/install-centos.sh | sudo bash
RHEL‑like: соберёт из исходников (занимает несколько минут).
# macOS (Apple Silicon)
curl -fsSL https://stream.centv.ru/install-macos.sh | bash
macOS: поставит бинарник в /usr/local/bin (попросит sudo при необходимости).

2) Первый запуск

Создайте пустой конфиг и запустите сервер на нужном порту (пример: 9060). 

sudo mkdir -p /etc/stream
sudo sh -c 'echo {} > /etc/stream/prod.json'

sudo /usr/local/bin/stream -c /etc/stream/prod.json -p 9060
После старта откройте Web UI: http://SERVER:9060

3) Вход в Web UI

Откройте http://SERVER:9060. Если вы запускаете впервые, система может создать пользователя по умолчанию (смотрите лог старта).

4) Создайте канал (Stream)

В Dashboard нажмите New Stream. Дальше:

  • Name — имя канала (как вы хотите видеть его в списке).
  • ID — короткий идентификатор (латиница/цифры, без пробелов). По нему строятся URL.
  • INPUT LIST → NEW INPUT — вставьте адрес источника (например udp://239.0.0.1:1234 или http://...).
  • OUTPUT LIST → NEW OUTPUT — добавьте выход. Для начала проще всего: UDP.

5) Проверьте, что поток выдаётся

Самый быстрый тест — открыть выход в плеере (например VLC/ffplay). Для HTTP‑TS обычно используется URL вида /live/<id>. 

# Пример: проверка HTTP-TS
ffplay -fflags nobuffer -flags low_delay -probesize 32 -analyzeduration 0 http://SERVER:9060/live/STREAM_ID

# Пример: проверка UDP
ffplay -fflags nobuffer -flags low_delay udp://239.0.0.1:1234
Если плеер не запускается — смотрите логи и статус входа в Dashboard.

6) Transcoding (опционально)

Транскодирование использует FFmpeg. Включайте его только если нужно поменять кодек/разрешение/битрейт или сделать несколько профилей. В Edit stream включите Enable transcoding и выберите профиль в вкладке Transcode.

7) Запуск как сервис (systemd)

Если хотите автозапуск после перезагрузки, зарегистрируйте шаблон сервиса и включите инстанс. 

sudo /usr/local/bin/stream --init

# Создать /etc/stream/prod.env (пример порта)
sudo sh -c 'echo STREAM_PORT=9060 > /etc/stream/prod.env'

# Включить и запустить
sudo systemctl enable --now stream@prod.service
Сервис читает конфиг: /etc/stream/prod.json

Если что-то не работает

  • Порт занят: выберите другой -p или остановите процесс, который слушает порт.
  • Нет сигнала: проверьте вход (битрейт/ошибки), сетевой интерфейс и multicast‑маршрутизацию.
  • Транскод не стартует: убедитесь, что ffmpeg установлен и доступен в PATH.
  • Забыли пароль: stream --reset-password (или -pass).

Streams (потоки)

Один поток = вход(ы) + правила обработки + выход(ы). Поддерживается backup/failover и переключение входов.

  • INPUT LIST: несколько источников, приоритет, быстрый порядок
  • OUTPUT LIST: несколько выходов на один поток
  • Analyze: быстрый анализ PIDs/битрейта/CC/PES ошибок

Входы

Основные типы зависят от сборки/модулей, но обычно доступны:

  • UDP / RTP
  • HTTP/HTTPS
  • RTSP / SRT
  • DVB (если есть /dev/dvb)

Выходы

Passthrough или транскодированный поток.

  • UDP
  • HLS / DASH
  • HTTP‑TS (/live/<id>)
  • Embed page (если включено)

Transcoding

Запускается через FFmpeg. Профили качества (CPU/NVENC), single profile или multi‑bitrate ladder.

  • Профили: 540p/720p/1080p, CPU/Nvidia
  • Логика ladder: для нескольких профилей — варианты (variants)
  • Audio Fix (если включено): исправление типовых проблем со звуком

Наблюдаемость и стабильность

Без лишней нагрузки: метрики по запросу или редкий rollup. Ограничения и перезапуски защищают инстанс.

  • Observability: CPU/MEM/Disk/Network + метрики потоков
  • Логи и события ошибок
  • Watchdog/limits: защита от перегрузки и restart‑storm

Access / Sessions

Контроль доступа к UI/API и просмотр клиентов.

  • Сессии клиентов
  • Токены/ACL (если включено)
  • Опция запуска: --no-web-auth

SoftCAM (опционально)

Интеграция descrambling в пайплайн. Опции задаются в input URL. Поведение зависит от конфигурации SoftCAM.

HLSSplitter / Buffer (опционально)

Доп. инструменты для распределения и буферизации потоков. Включаются в UI настройками интерфейса.
