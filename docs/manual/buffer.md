# Buffer (HTTP‑TS буфер)

Buffer — это встроенный HTTP‑сервер, который:

- читает вход (обычно HTTP‑TS),
- держит буфер на стороне сервера,
- и отдаёт TS клиентам стабильно.

Это удобно, когда источник “рвётся” или отдаёт поток рывками.

## Как устроено

- **Resource** — один буфер/ресурс (по сути “канал” внутри Buffer).
- У ресурса может быть несколько входов (primary/backup).

## Быстрый старт

1. Откройте вкладку **Buffer**.
2. Нажмите **NEW RESOURCE**.
3. Задайте:
   - Path (как будет выглядеть URL),
   - вход(ы) ресурса.
4. Сохраните.
5. Откройте output URL ресурса и проверьте в плеере.

## Когда помогает

- IPTV панель отдаёт “всплесками” (burst).
- На WAN бывают короткие провалы.
- Нужно, чтобы у клиентов было меньше зависаний.

## Если не работает

- проверьте глобальные настройки Buffer в **Settings → Buffer**,
- проверьте входной URL с сервера (curl),
- посмотрите ошибки в **Log**.
