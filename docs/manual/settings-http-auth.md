# Settings → HTTP Authentication

Этот раздел отвечает за защиту Web UI и API.

## Включить/выключить

- **Enable HTTP authentication** — общий переключатель.

## Способы доступа

В зависимости от ваших настроек можно использовать:

- Basic auth по пользователям (таблица Users),
- allowlist/denylist по IP,
- статические токены,
- внешний backend (on‑play / on‑publish), если вы интегрируете авторизацию со своей системой.

## Allowed / Denied IPs

Полезно, если UI/API должны открываться только из вашей сети.

Примеры:

```text
127.0.0.1, 10.0.0.0/24
```

## Tokens

Статические токены — простой способ дать доступ без логина/пароля.
Подходит для тестов или внутренних клиентов.

## Если вы отключаете авторизацию

Есть CLI флаг:

```bash
sudo /usr/local/bin/stream --no-web-auth -c /etc/stream/prod.json -p 9060
```

!!! danger "Опасно"
    Используйте только в закрытой сети. Это отключает защиту Web UI.
