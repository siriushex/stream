# Settings → HTTP Play

HTTP Play — это “публичные ссылки” для просмотра и тестов.
Обычно это:

- `/play/...` (просмотр),
- `/live/...` (HTTP‑TS),
- плейлисты.

## Allow HTTP access to all streams

Если включено, Stream Hub разрешает HTTP доступ к потокам (по правилам вашей авторизации/allowlist).

Если вы не хотите случайно открыть всё наружу — держите выключенным и ограничивайте доступ.

## HTTP play port

Отдельный порт для “play” трафика.
Полезно, когда:

- UI/API на одном порту,
- а клиентов хочется вынести на другой (проще firewall и лимиты).

## HTTP buffer size (KB)

Буфер на стороне сервера для HTTP клиентов.

- больше буфер → меньше “дёрганий” у клиента,
- но больше RAM на соединения.

## Логотипы и скриншоты

Если вы используете плейлисты/портал, можно указать пути к:

- логотипам каналов,
- скриншотам.

## Если что‑то не играет

1. Проверьте вход (ONLINE, битрейт).
2. Проверьте доступ к порту/URL с клиентской машины.
3. Посмотрите **Sessions** и **Log**.
