# Входы (Input)

Вход — это адрес источника (откуда Stream Hub берёт поток).

## Шаги

1. Откройте канал (Edit stream).
2. В **INPUT LIST** нажмите **NEW INPUT**.
3. Вставьте адрес.
4. Нажмите **OK**, потом **Save**.

## Какие входы бывают

Зависит от вашей задачи, но чаще всего используют:

- **UDP / RTP** — multicast/unicast в локальной сети.
- **HTTP‑TS** — поток по HTTP.
- **HLS** — плейлист и сегменты.
- **DVB** — приём с DVB‑адаптеров (если они есть на сервере).

## Примеры адресов

```text
udp://239.0.0.1:1234
udp://enp5s0f1@239.0.0.1:1234
http://example.com:8000/live/channel
hls://example.com/live/index.m3u8
dvb://...
```

!!! tip "Если не уверены"
    Начните с самого простого адреса. Если нужно выбрать интерфейс — добавьте `iface@`.

## Параметры после `#`

Иногда к адресу добавляют параметры в конце, после `#`.
Например: синхронизация, выбор сервиса, детекторы качества.

Подробнее: [Форматы адресов](address-format.md)

## Как понять, что вход живой

- В карточке появился битрейт.
- Статус стал ONLINE.

## Несколько входов (Backup / failover)

Вы можете добавить 2+ входа в один канал.
Тогда Stream Hub может переключаться на резерв, если основной пропал.

Где настраивается:

- вкладка **Backup** в окне канала,
- и/или приоритет входов в **INPUT LIST** (порядок важен).

## Устойчивость для HTTP/HLS (если сеть “плохая”)

Если источник по HTTP/HLS иногда подвисает:

- увеличьте таймауты (в Input settings),
- и включите сетевой профиль (если он доступен в вашем UI), например `wan` или `bad`.

Если вам удобнее через URL, параметры выглядят так:

```text
http://host/stream.ts#net_profile=bad
```

Не включайте “тяжёлые” режимы без необходимости — это увеличивает буферы и задержку.

## Quality detectors

Для мониторинга качества входа можно включить датчики:

- CC errors,
- no audio,
- stop video,
- AV desync,
- silence detect.

См. отдельную страницу: [Quality detectors](quality-detectors.md)

## Если вход не работает

- Проверьте адрес. Ошибка в одном символе — и сигнала не будет.
- Для multicast проверьте сеть и интерфейс.

## Create radio (аудио + PNG → UDP TS)

Если у вас есть только аудио поток (радио) — можно сделать “TV‑совместимый” MPEG‑TS:

- откройте **Input settings**,
- выберите **Type: Create radio**,
- укажите **Audio URL** + PNG,
- нажмите **Start**,
- сохраните канал (**Save**).

Детально: [Create radio](create-radio.md)
