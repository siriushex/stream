# Логи

Логи — самый быстрый способ понять “почему не работает”.

## Где смотреть

- В UI: вкладка **Log**.
- В systemd: `journalctl`.
- В файл: если вы запускали с `--log /path/to/file.log`.

## Если запускали из консоли

Логи уже в терминале.

## Если запускали как systemd‑сервис

```bash
journalctl -u stream@prod.service -f
```

## Полезно при отладке

Проверьте, слушает ли порт:

```bash
ss -lntp | grep ':9060' || true
```

## Что искать в логах

- ошибки входа (таймаут, нет данных),
- сообщения про ffmpeg (если включён транскод/Audio Fix),
- отказ в доступе (auth / allow/deny),
- ошибки записи на диск (HLS/DASH, если включено).
