# Settings → Edit Config / Config History / Restart / License

Эти разделы нужны, когда вы работаете “на низком уровне” с конфигом и обновлениями.

## Edit Config

Это редактор полного конфига (settings + users + streams + adapters + services).

Режимы:

- **Merge** — аккуратно добавить/обновить.
- **Replace** — заменить всё (осторожно).

!!! danger "Replace"
    Если ошибиться в JSON, можно “положить” конфиг. Делайте бэкап перед правками.

## Config History

История ревизий конфига.
Удобно, когда нужно:

- понять, кто и что менял,
- откатиться на рабочую ревизию.

## Restart (Reload)

Кнопка **Reload (safe)** делает мягкую перезагрузку конфигурации.
Обычно это безопаснее, чем полный рестарт процесса.

## License

Показывает текущую информацию о лицензии (если она используется в вашей сборке).
