# Observability

Observability — это страница с метриками сервера и каналов.
По умолчанию она может быть выключена, чтобы не тратить лишние ресурсы.

## Как включить

1. Откройте **Settings**.
2. Найдите блок **Observability**.
3. Включите **Enabled**.
4. Откройте вкладку **Observability** в верхнем меню.

## Как пользоваться (быстро)

На странице обычно есть:

- **Range** — за какой период строить графики (например 24h/7d/30d).
- **Scope**:
  - **Global** — общий обзор по серверу.
  - **Stream** — графики по одному выбранному каналу.
- **Stream** — выбор канала (активно, когда Scope = Stream).
- **Refresh** — обновить данные (без постоянного опроса по умолчанию).

## Что вы увидите

### Server

- CPU / Memory / Disk.
- Network RX/TX (по выбранному интерфейсу).

### Streams (Global)

- Total bitrate (сводка по потокам).
- Streams on air / down.
- Switches / alerts (если включено).

### Stream (канал)

Когда выбран конкретный Stream, обычно доступны графики:

- битрейт,
- ошибки по входу (например CC/PES), если они собираются,
- переключения входов (если есть Backup).

!!! tip "Если графики пустые"
    Проверьте, что Observability включена в Settings, и нажмите Refresh.
