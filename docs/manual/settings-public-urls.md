# Settings → Public URLs

Public URLs — это “какие ссылки показывать пользователю”.

Stream Hub генерирует ссылки для HLS/HTTP‑TS/Embed автоматически.
Чтобы они были правильные (особенно за NAT / за reverse‑proxy), задайте публичный адрес.

## Когда нужно

- сервер стоит за NAT, и внешний IP/порт отличается;
- Web UI открыт через домен и прокси (например `https://tv.example.com`);
- вы хотите, чтобы кнопки Copy/Play копировали “внешний” URL.

## Как проверить

1. Откройте любой канал.
2. В OUTPUT LIST выберите HLS или HTTP‑TS.
3. Скопируйте URL и откройте с клиентской машины.

Если ссылка ведёт на “не тот” хост/порт — исправляйте Public URLs.

## Какие поля бывают

Обычно достаточно заполнить:

- **HTTP base URL** (например `http://example.com:9060`)

Опционально:

- **HTTPS base URL** — если UI/выходы доступны по HTTPS.
- **HLS base URL** / **DASH base URL** — если HLS/DASH раздаются с другого домена.
- **RTMP base URL** / **RTSP base URL** — если вы генерируете push‑URL шаблонами.
