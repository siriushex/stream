# Backup / failover (резервный вход)

Backup нужен, когда у канала есть несколько источников.
Если основной вход “упал”, Stream Hub может переключиться на резервный.

## Как включить

1. В канале добавьте 2+ входа в **INPUT LIST**.
2. Откройте вкладку **Backup**.
3. Выберите **Backup mode** и сохраните.

Если Backup mode пустой, но входов больше одного — обычно включается активный режим по умолчанию.

## Режимы (простыми словами)

- **Disabled** — никаких переключений.
- **Passive** — держим только активный вход, резервные поднимаем при проблеме.
- **Active** — можем держать несколько входов “тёплыми”, чтобы переключение было быстрее.
- **Active + stop if all inactive** — как Active, но если все входы неактивны долго, можно “успокоиться” и остановить попытки (полезно при массовом падении источников).

## Важные таймауты

- **No data timeout** — сколько ждать данных, прежде чем считать вход “плохим”.
- **Stable OK** — сколько секунд вход должен быть стабильным, чтобы считать его “нормальным”.
- **Probe interval** — как часто перепроверять (если включено).

## Как проверить

1. Запустите канал и убедитесь, что он ONLINE.
2. Временно отключите основной источник.
3. Посмотрите, что **Active input** переключился на другой вход.
4. Верните основной и проверьте возврат (если включён).

!!! tip "Чтобы не было “флаппинга”"
    Увеличьте Stable OK и Return delay. Тогда Stream Hub не будет дёргаться между входами.
