# Create radio (аудио + PNG → UDP TS)

Функция для радио‑каналов: аудио поток + статичная картинка,
на выходе получается MPEG‑TS по UDP.

## Где находится

**Edit stream → Transcode → Create radio**

## Коротко

- Указываете **Audio URL** (например Icecast/ICY MP3).
- Выбираете PNG.
- Нажимаете **Start**.
- Получаете UDP‑поток, который можно использовать как вход.

## Шаги

1. Откройте канал → **Edit stream**.
2. Перейдите в **Transcode**.
3. В блоке **Create radio** заполните:
   - **Audio URL**
   - **PNG file**
   - **Output UDP URL**
4. Нажмите **Start**.
5. Нажмите **Set as stream input** (если нужно сразу использовать).

## Важные параметры

- **Use CURL pipe**  
  Рекомендуется для ICY/MP3 потоков.  
  Стабильнее, чем прямой вход ffmpeg.

- **Video settings**  
  Обычно достаточно дефолтов:  
  `25 fps`, `270x270`, `libx264`, `veryfast`, `yuv420p`.

- **Audio settings**  
  По умолчанию: `AAC 256k / 48 kHz / 2ch`.

- **TS/UDP**  
  `pkt_size=1316` и `muxdelay=0.7` — рабочие значения по умолчанию.

## Статус и логи

- **Running / Stopped / Error**
- Есть лог‑хвост для диагностики (последние строки).

Если процесс упал — можно нажать **Restart**.

## Ошибки

- **ffmpeg не найден**  
  Установите `ffmpeg`.

- **Audio URL не отвечает**  
  Проверьте доступность и таймауты.

- **PNG не найден**  
  Проверьте путь и права.

!!! tip "Где использовать"
    Create radio удобно как “живой резерв” — можно включить в Backup,
    или подменять основной вход на лету.
